Kritik bulgu: Mimari dokümanları, gerçek sistemle uyuşmuyor

MHEWS_architecture_description.md (IEEE 42010) ve MHEWS_software_design_description.md, sistemi Django + DRF + Celery + Redis + PostgreSQL yığını üzerinde 12 ayrı Django app olarak tanımlıyor. Ama iş_planı_istereler.txt'deki 38 spec kaydında (gerçek migration'lar, gerçek tablo/fonksiyon isimleri) Django, Celery veya Redis'e tek bir referans bile yok — buna karşılık 49 yerde Supabase/Edge Function/Deno/RLS/PostGIS/Vue geçiyor. Sistem gerçekte tamamen farklı bir teknoloji yığını üzerinde inşa edilmiş.

Mimari dokümanı (planlanan, muhtemelen terk edildi)
  • Django 5.x + DRF, 12 ayrı Django app
  • Celery 5 + Celery Beat (görev kuyruğu)
  • Redis 7 (broker + cache + SSE)
  • django-guardian (ABAC), simplejwt
  • Dual profil: Cloud + On-premise (Ollama LLM)
Gerçekte inşa edilen (iş planından doğrulandı)
  • Supabase (PostgreSQL 16 + PostGIS + RLS)
  • Deno Edge Functions (iş mantığı, dispatch)
  • pg_net + pg_cron (DB içi tetikleme/zamanlama)
  • Node.js aggregator (GEWS, ayrı konteyner)
  • Vue 3 + Vite frontend, tek bulut profili

Aşağıdaki diyagramın tamamı sağ sütuna — yani doğrulanmış gerçek duruma — dayanıyor.

Roller & Erişim

4 sabit kademe · RLS ile uygulanıyor

Roller Supabase RLS politikalarıyla veritabanı seviyesinde uygulanıyor; SRS'te planlanan "custom permissions" (konfigüre edilebilir roller) henüz yok — 4 kademe sabit.

viewer
Salt okuma. Afet Ansiklopedisi, harita, genel görünümler. CAP taslağı göremez (viewer görünürlüğü daraltıldı).
org_admin
Kendi organizasyonuna atanmış onaylı topluluk bildirimlerini salt-okunur görür. Onay/red yetkisi yok.
country_admin
Ülke kapsamlı CRUD: kişi rehberi, kaynak yönetimi, topluluk bildirimi moderasyonu (onay/red).
super_admin
Küresel kapsam: hazard taksonomisi, harita katmanları (map_layers), kullanıcı askıya alma, denetim (audit) sekmesi.

Kimlik doğrulama: TOTP tabanlı MFA (SRS NFR-0057, CRITICAL) — herkese açık "Hesap Güvenliği" sayfası, AAL tabanlı login challenge, özel mfa_recovery_codes tablosu (Supabase'de native karşılığı yok). Self-registration bilinçli olarak tamamen kaldırılmış — sadece davetle onboarding.
